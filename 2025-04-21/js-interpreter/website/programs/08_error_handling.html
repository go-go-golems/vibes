Program Code

This program tests error handling mechanisms in JavaScript: try-catch-finally blocks, custom errors, and error type checking.

// Error handling
try {
    // This will throw an error
    const result = 10 / 0;
    console.log("Result:", result);
    
    // This line won't execute
    console.log("This won't be printed");
} catch (error) {
    console.log("Caught an error:", error.message);
} finally {
    console.log("Finally block always executes");
}

// Try-catch with custom error
try {
    const age = 15;
    if (age < 18) {
        throw new Error("Age must be at least 18");
    }
    console.log("Access granted");
} catch (error) {
    console.log("Access denied:", error.message);
}

// Error handling with specific error types
try {
    // Trying to access property of undefined
    const user = undefined;
    const name = user.name;
} catch (error) {
    if (error instanceof TypeError) {
        console.log("Type error:", error.message);
    } else {
        console.log("Other error:", error.message);
    }
}

Analysis

The program successfully executed all error handling mechanisms, but with some interesting behavior:

  • Division by zero: Unlike some JavaScript environments that would throw an error, Goja returns "+Inf" (positive infinity) for division by zero, which is actually consistent with the IEEE 754 floating-point standard
  • Finally block: Correctly executed the finally block
  • Custom error: Successfully threw and caught a custom error with the appropriate message
  • Error type checking: Correctly identified the TypeError when trying to access a property of undefined

This confirms that the Goja interpreter correctly implements error handling mechanisms in JavaScript, including try-catch-finally blocks, custom errors, and error type checking. The only notable difference is in the handling of division by zero, which returns infinity rather than throwing an error.
